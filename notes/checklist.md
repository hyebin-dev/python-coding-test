# 제출 전 체크리스트

제출 직전에 10~20초만 보고 실수로 틀리는 것을 막는다.

---

## 입력/출력
- 입력 많으면 sys.stdin.readline 사용했나?
- 출력 많으면 리스트에 모아 "\n".join() 했나?
- strip()을 너무 써서 공백이 필요한데 날리진 않았나?

## 자료형/정수
- Python은 big int지만, 시간초과를 부르는 비효율 연산(문자열 덧붙이기 반복 등) 없는지?
- float 비교가 필요한 문제면 오차 처리를 했나? (eps, round 등)

## 인덱스/경계
- 0-index/1-index 섞이지 않았나?
- 빈 배열/원소 1개/최솟값/최댓값 케이스 고려했나?
- 슬라이싱이 O(k)라 반복문 안에서 남발하고 있지 않나?

## 시간복잡도/자료구조
- list in / remove / pop(0) 같은 O(N) 연산을 반복하고 있지 않나?
- deque가 필요한데 list로 큐를 구현하고 있지 않나?
- set/dict로 바꿔야 하는데 리스트 탐색을 하고 있지 않나?

## BFS/DFS
- BFS면 deque 사용했나?
- visited 처리 시점이 enqueue 시점인가? (중복 enqueue 방지)
- DFS 재귀면 recursionlimit 올렸나? (필요할 때만)
- 그래프 방향/무방향 간선 추가가 맞나?

## DP
- dp 초기값(불가능 상태)을 명확히 했나? (inf, -inf, None 등)
- 점화식 인덱스 범위(음수/초과) 안전한가?
- 모듈러 문제면 매 연산마다 mod 적용했나?

## 이분탐색
- bisect 사용 시 left/right 의미를 정확히 이해했나?
- 답 탐색이면 check가 단조성 만족하나?
- lo/hi 초기값이 문제 정의에 맞나?

## 구현/시뮬레이션
- "동시에 바뀌는 상태"를 순서대로 덮어써서 오염시키지 않았나?
- deepcopy 남발로 시간초과 유발하지 않았나?

## 최종 확인
- 예제 입력을 직접 돌려 예제 출력과 정확히 일치하나?
- 내가 만든 반례 1~2개로 다시 확인했나?
